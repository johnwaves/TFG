## Casos de uso

| **Caso de uso**                      | **Descripción detallada**                                                                                                                                                                                                                                                                              | **Requisitos satisfechos** |
|--------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------|
| **CU01: Registro de pacientes**      | **Descripción:** El personal sanitario podrá registrar y actualizar los datos de pacientes en el sistema. Además, se incluirá la opción de añadir la información del tutor o acompañante en caso de que el paciente lo requiera. <br> **Precondiciones:** Usuario autenticado como farmacéutico o técnico de farmacia. <br> **Postcondiciones:** Paciente registrado en el sistema. <br> **Flujo principal:** <ol><li>El usuario selecciona la opción de registrar paciente.</li><li>El usuario ingresa los datos del paciente.</li><li>El sistema valida la información.</li><li>El sistema guarda los datos del paciente.</li></ol> | RF1, RFN1, RFN4             |
| **CU02: Dispensación de medicamentos** | **Descripción:** El sistema registrará la dispensación de medicamentos, incluyendo detalles como fecha, hora e identidad del personal que realizó la dispensación. <br> **Precondiciones:** Prescripción válida. <br> **Postcondiciones:** Registro de dispensación almacenado. <br> **Flujo principal:** <ol><li>El usuario selecciona la opción de dispensar medicamento.</li><li>El usuario ingresa la prescripción y la identifica con el paciente.</li><li>El sistema valida la prescripción.</li><li>El sistema guarda el registro de dispensación.</li></ol> | RF2, RFN3, RFN6             |
| **CU03: Análisis de adherencia**     | **Descripción:** El sistema permitirá al personal sanitario realizar el seguimiento de los tratamientos de los pacientes para determinar la adherencia. Se podrán registrar observaciones, ingresar datos de seguimiento como constantes vitales y generar alertas de seguimiento. <br> **Precondiciones:** Paciente registrado en el sistema. <br> **Postcondiciones:** Nivel de adherencia calculado y registrado. <br> **Flujo principal:** <ol><li>El usuario selecciona al paciente y el tratamiento a seguir.</li><li>El usuario ingresa los datos de seguimiento (vitales, somatométricos, etc.).</li><li>El sistema genera alertas o recordatorios si es necesario.</li><li>El sistema calcula y muestra el nivel de adherencia.</li></ol> | RF3, RF4, RFN5, RFN7        |
| **CU04: Consulta de información médica** | **Descripción:** El sistema permitirá a los actores consultar información sobre tratamientos y medicamentos, accesible de manera segura y según los permisos correspondientes. <br> **Precondiciones:** Usuario autenticado con permisos de consulta. <br> **Postcondiciones:** Información médica consultada. <br> **Flujo principal:** <ol><li>El usuario inicia sesión y selecciona la opción de consulta.</li><li>El sistema verifica los permisos del usuario.</li><li>El usuario selecciona el tipo de información a consultar (tratamientos, medicamentos).</li><li>El sistema muestra la información solicitada.</li></ol> | RF4, RFN1, RFN2, RFN4       |
| **CU05: Automatización de tareas administrativas** | **Descripción:** El sistema automatizará ciertas tareas administrativas, como la generación de informes y la programación de seguimientos, mejorando la eficiencia y reduciendo la carga de trabajo manual. <br> **Precondiciones:** Usuario autenticado con permisos de administrador. <br> **Postcondiciones:** Tareas administrativas automatizadas. <br> **Flujo principal:** <ol><li>El usuario selecciona la opción para generar informes o programar seguimientos.</li><li>El sistema recopila y procesa la información necesaria.</li><li>El sistema genera el informe o programa el seguimiento automáticamente.</li><li>El usuario recibe una notificación del proceso completado.</li></ol> | RF5, RFN3, RFN5, RFN6       |
| **CU06: Gestión de usuarios del sistema** | **Descripción:** El administrador del sistema podrá gestionar las cuentas de usuarios, incluyendo la creación, modificación y eliminación de cuentas, así como la asignación de roles y permisos. <br> **Precondiciones:** Usuario autenticado como administrador. <br> **Postcondiciones:** Cambios en las cuentas de usuario aplicados. <br> **Flujo principal:** <ol><li>El administrador selecciona la opción de gestión de usuarios.</li><li>El administrador crea, modifica o elimina una cuenta de usuario.</li><li>El sistema valida los cambios y los guarda.</li><li>El usuario afectado recibe una notificación de los cambios.</li></ol> | RF6, RFN1, RFN7             |
| **CU07: Seguimiento de tratamientos no farmacológicos** | **Descripción:** El sistema permitirá registrar y realizar el seguimiento de tratamientos no farmacológicos, como hábitos de vida saludables. Se generarán alertas y recordatorios para el cumplimiento de estos tratamientos. <br> **Precondiciones:** Paciente registrado y tratamiento asignado. <br> **Postcondiciones:** Seguimiento del tratamiento registrado. <br> **Flujo principal:** <ol><li>El usuario selecciona el paciente y el tratamiento no farmacológico.</li><li>El usuario ingresa o consulta los datos de seguimiento.</li><li>El sistema genera alertas o recordatorios si es necesario.</li><li>El sistema registra el progreso del paciente.</li></ol> | RF7, RFN1, RFN3, RFN4       |
